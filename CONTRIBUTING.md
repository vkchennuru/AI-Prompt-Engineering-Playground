## Ways to Contribute

### üêõ Report a Bug
If something is broken or not working as expected:
1. Go to the **Issues** tab on GitHub
2. Click **New Issue**
3. Describe the problem, which page/level it's on, and your browser
4. Add the label `bug`

### üí° Suggest an Improvement
Have an idea for a new exercise, quiz question, or content section?
1. Open a **New Issue**
2. Describe your idea clearly
3. Add the label `enhancement`

### üìù Improve Content
To fix typos, improve explanations, or add new content:
1. **Fork** this repository (click Fork at the top right)
2. Make your changes in your fork
3. Submit a **Pull Request** with a clear description of what you changed and why

### üåç Translate to Another Language
We welcome translations! To translate:
1. Fork the repository
2. Create a new folder named with the language code (e.g., `hi/` for Hindi, `te/` for Telugu)
3. Copy and translate the HTML files
4. Submit a Pull Request

### ‚ûï Add a New Level or Topic
If you want to add an entirely new level or a major new section:
1. Open an Issue first to discuss the idea
2. Once approved, fork and implement
3. Follow the existing file structure and style conventions
4. Submit a Pull Request

---

## Code Style Guidelines

- Keep HTML, CSS, and JS self-contained per file where possible
- Shared styles go in `shared.css`, shared logic in `shared.js`
- Use the existing color variables (e.g., `var(--l1)` through `var(--l6)`)
- All interactive exercises should work without an internet connection
- Keep the tone beginner-friendly and encouraging

---

## Pull Request Checklist

Before submitting a PR, please confirm:
- [ ] Content is accurate and tested
- [ ] No broken links or missing files
- [ ] Pages render correctly on mobile and desktop
- [ ] JavaScript functions work in Chrome and Firefox
- [ ] You have added your name to contributors (optional but appreciated)
